# Как разместить лендинг на GitHub Pages — пошагово, с нуля

Эта инструкция рассчитана на то, что ты делаешь это впервые. Иди строго сверху вниз, не пропуская шаги. В конце сайт будет доступен по адресу вида `https://vladislavshter.github.io/egregor-landing/`.

---

## ✅ Хорошая новость про видео

Видео Egregor берётся прямо с твоего YouTube (`https://youtu.be/y8oZdDBQYhc`) и уже встроено в лендинг. Поэтому тяжёлый файл загружать НЕ нужно.

На GitHub ты загрузишь всего **3 лёгких файла**:
- `index.html` — сам лендинг
- `1.gif` — анимация интерфейса
- `demo-poster.jpg` — картинка-обложка

Все три маленькие, грузятся через браузер без проблем. Файл `Egregor1.mp4` тебе больше не нужен — игнорируй его.

---

## ЧАСТЬ 1. Аккаунт GitHub

У тебя уже есть аккаунт `github.com/VladislavShter` — значит, эта часть пропускается. Просто войди в него на `https://github.com`.

(Если бы аккаунта не было: `github.com` → Sign up → email, пароль, username → подтвердить email.)

---

## ЧАСТЬ 2. Создание репозитория (это «папка» сайта на GitHub)

1. Войди в аккаунт на `https://github.com`.
2. В правом верхнем углу нажми **«+»** → **New repository**.
   (Или открой ссылку `https://github.com/new`)
3. Заполни:
   - **Repository name**: впиши `egregor-landing`
     (без пробелов и заглавных букв; это имя попадёт в адрес сайта)
   - **Public / Private**: обязательно **Public** ⚠️
     (бесплатный GitHub Pages работает только с публичными репозиториями)
   - Галочку **Add a README file** — НЕ ставь.
   - Остальное не трогай.
4. Нажми зелёную **Create repository**.
5. Откроется пустой репозиторий — переходи к ЧАСТИ 3.

---

## ЧАСТЬ 3. Загрузка трёх файлов

1. На странице репозитория нажми **Add file** → **Upload files**
   (или ссылку **uploading an existing file** в тексте).
2. Перетащи в окно три файла: `index.html`, `1.gif`, `demo-poster.jpg`.
   (Или нажми **choose your files** и выбери их.)
3. Дождись, пока все три прогрузятся (зелёные галочки рядом с именами).
4. Внизу в поле **Commit changes** впиши что угодно, например `add landing`.
5. Нажми зелёную **Commit changes**.
6. Файлы появятся в репозитории. Убедись, что `index.html` лежит **в корне** (в общем списке, не внутри папки).

> ⚠️ Имена файлов должны быть ТОЧНО: `index.html`, `1.gif`, `demo-poster.jpg` — с учётом регистра. Иначе картинки не подгрузятся.

---

## ЧАСТЬ 4. Включение GitHub Pages (репозиторий → сайт)

1. В репозитории нажми вкладку **Settings** (вверху справа в меню репозитория).
2. В левом меню найди и нажми **Pages**.
3. В разделе **Build and deployment**:
   - **Source**: выбери **Deploy from a branch**.
   - **Branch**: выбери **main**, папку оставь **/ (root)**.
   - Нажми **Save**.
4. Подожди 1–2 минуты, обнови страницу.
5. Появится зелёная надпись:
   **"Your site is live at https://vladislavshter.github.io/egregor-landing/"**
6. Нажми на адрес или **Visit site** — откроется твой лендинг. 🎉

> Первая публикация иногда занимает 2–5 минут. Ошибка 404 в первые минуты — нормально, просто подожди и обнови.

---

## ЧАСТЬ 5. Проверка (обязательно!)

Открой свой сайт и проверь:
- [ ] Открывается, фон тёмный с неоновым свечением.
- [ ] Переключаются 3 вкладки сверху (Egregor / SovereignWeb3 Browser / Sovereign).
- [ ] Меняются цвета при переключении вкладок.
- [ ] Кнопки RU / EN слева переключают язык.
- [ ] На вкладке Egregor крутится gif.
- [ ] Видео с YouTube показывается и играет.
- [ ] Адреса кошельков (0x… и T…) и кнопки Copy работают.
- [ ] Кнопки «Более подробно» ведут на репозиторий Egregor.
- [ ] Открой сайт на ТЕЛЕФОНЕ — текст и картинки не наползают.

Если что-то не так — сделай скриншот и пришли мне.

---

## ЧАСТЬ 6. Как редактировать сайт потом

1. Открой репозиторий → нажми на файл `index.html`.
2. Нажми значок **карандаша** (Edit this file) вверху справа.
3. Найди текст (Ctrl+F помогает), поменяй.
4. Внизу нажми **Commit changes**.
5. Через 1–2 минуты правка появится на сайте.

> Меняй только текст между `>` и `<`. Теги в угловых скобках `< >` не трогай. Если что-то сломается — история версий на GitHub позволяет откатить назад.

---

## ЧАСТЬ 7. (На будущее) Свой домен

Когда купишь домен:
1. Settings → Pages → **Custom domain** → впиши домен → Save.
2. У регистратора домена пропиши DNS-записи, которые покажет GitHub.
3. Готово. Сам `index.html` менять не нужно.

---

## Короткая шпаргалка

1. `github.com/new` → имя `egregor-landing`, **Public**, без README → Create.
2. **Add file → Upload files** → перетащить `index.html`, `1.gif`, `demo-poster.jpg` → **Commit changes**.
3. **Settings → Pages → Deploy from branch → main / root → Save**.
4. Подождать 2 мин → открыть `https://vladislavshter.github.io/egregor-landing/`.

Когда сайт будет живой — пришли ссылку, проверим вместе.
